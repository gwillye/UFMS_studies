# 🎓 UFMS Studies

A consolidated home for my **Computer Science / Information Systems coursework and academic projects at UFMS** — grouped by area instead of scattered across "bucket" repos. Each subfolder is a self-contained project with its own README and instructions.

> Academic portfolio. Code is my own; client/professional projects live in their own dedicated repositories.

## 📚 Projects
| Folder | Area | What it is |
|---|---|---|
| [`data-structures/`](data-structures) | Algorithms & Data Structures | Trees, heaps and sorting in **C/C++**. *Roadmap: grow into a small executable application built on these structures.* |
| [`database-lab/`](database-lab) | Databases | SQL modeling (DDL/DML, queries, triggers) with **PostgreSQL + MySQL via Docker**. *Roadmap: elevate from coursework into a complete, improved database project.* |
| [`distributed-computing/`](distributed-computing) | Distributed Systems | A **Flask** MVC web service (Flask-Admin, SQLAlchemy, HTTP Basic Auth) running on **Docker**. |
| [`web-programming/`](web-programming) | Web | Front-end exercises (HTML/CSS/JS) plus a small **React** app. |
| [`quiz-app/`](quiz-app) | Web | A multi-page **quiz platform** (create / play / ranking) in vanilla HTML/CSS/JS. |

## 🛠️ Tech across the projects
C · C++ · SQL (PostgreSQL / MySQL) · Docker · Python (Flask) · JavaScript · React · HTML/CSS

## ▶️ Running a project
Each subfolder has its own `README.md` with setup and run instructions (compile commands for C/C++, `docker compose` for the DB and Flask projects, a static server for the web ones).
